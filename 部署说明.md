## 🚀 第一步：部署到 Vercel

### 1.1 打开 Vercel 网站

在浏览器中打开：https://vercel.com

如果没有账号，点击 "Sign Up" 用 GitHub 账号注册。

---

### 1.2 导入项目

1. 登录后，点击页面上的 **"Add New"** 按钮
2. 选择 **"Project"**
3. 点击 **"Continue with GitHub"** 按钮

---

### 1.3 授权 GitHub

1. 如果是第一次使用，会提示授权 Vercel 访问你的 GitHub
2. 点击 **"Authorize Vercel"** 按钮
3. 选择 **"All repositories"** 或只选择 `yaoyao` 仓库
4. 点击 **"Install"**

---

### 1.4 选择仓库

在仓库列表中找到并点击 **"yaoyao"** 仓库（如果看不到，点击 "Adjust GitHub App Permissions" 确保权限正确）

---

### 1.5 配置项目设置

在导入页面填写以下配置：

| 配置项 | 填写内容 |
|--------|----------|
| **Project Name** | `yaoyao` |
| **Framework Preset** | 选择 `Other` |
| **Root Directory** | 留空或填写 `./` |
| **Build Command** | 留空 |
| **Output Directory** | 留空或填写 `./` |
| **Install Command** | 留空 |

其他选项保持默认即可。

---

### 1.6 开始部署

点击页面底部的 **"Deploy"** 按钮

---

### 1.7 等待部署完成

- 部署过程约需 **1-2 分钟**
- 页面会显示部署进度
- 看到 "Congratulations!" 页面表示部署成功 ✅

---

### 1.8 记录 Vercel 链接

部署成功后，Vercel 会提供访问链接：
```
https://yaoyao.vercel.app
```

**⬇️ 先把这个链接复制下来，后面要用。**

---

## 📄 第二步：配置 GitHub Pages

### 2.1 打开仓库设置

在浏览器中打开：
```
https://github.com/qjianxun750-stack/yaoyao/settings/pages
```

如果提示登录，使用你的 GitHub 账号登录。

---

### 2.2 配置 Pages 设置

在页面中找到 **"Source"** 部分：

1. **Source**：选择 `Deploy from a branch`
2. **Branch**：选择 `main`
3. **Folder**：选择 `/ (root)`

设置完成后点击 **"Save"** 按钮。

---

### 2.3 等待部署

- 页面会显示 "Your site is live at..."
- 部署约需 **1-2 分钟**
- 看到 ✅ 图标表示部署成功

---

### 2.4 记录 GitHub Pages 链接

部署成功后，页面顶部会显示链接：
```
https://qjianxun750-stack.github.io/yaoyao/
```

**⬇️ 也把这个链接复制下来。**

## ❓ 常见问题

### Q1: Vercel 部署失败怎么办？
**A**: 检查以下几点：
- 确保选择了正确的仓库
- 确保 Root Directory 设置为 `./`
- 查看错误日志，根据提示调整

### Q2: GitHub Pages 显示 404？
**A**:
- 确保分支选择为 `main`
- 确保文件夹选择为 `/ (root)`
- 等待 2-3 分钟后再刷新

### Q3: 页面能打开但功能不正常？
**A**:
- 打开浏览器控制台（F12）查看是否有错误
- 确保所有 JavaScript 文件正确加载
- 清除浏览器缓存后重试

### Q4: 数据统计没有数据？
**A**:
- Umami: 需要至少一次访问才会开始记录
- Supabase: 需要至少摇一次骰子或生成一次分享才会有数据
